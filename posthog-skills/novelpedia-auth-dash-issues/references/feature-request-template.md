# Feature Request Template

Use this template when filing `type:feature` or `type:epic` issues on Inkosei/auth-dash.

```
**Feature:**
What you want built. Be specific about the user outcome, not the implementation.

**Motivation:**
Why does this matter? Who wants it and why? Link to any customer requests or data.

**Proposed Solution:**
How you imagine it working. If you have thoughts on implementation, note them here.
This is a suggestion, not a constraint -- the team may find a better approach.

**Alternatives Considered:**
Other approaches that were rejected and why.

**Success Criteria:**
How do we know when this is done? What metrics, behaviors, or outcomes define success?

**Scope:**
- [ ] Frontend only
- [ ] Backend only
- [ ] Full-stack (frontend + backend)
- [ ] Infra/DevOps

**Assignee:** @username
**Milestone:** Phase N: ...
**Labels:** type:feature, P:x, area:backend|frontend|infra
```

## Epic Template (for large features)

For `type:epic` items that will be broken into multiple issues:

```
**Epic:**
High-level description of the feature area

**Goal:**
What business or user outcome does this epic serve?

**Phases:**
Breakdown of how this epic will be delivered across milestones.

**Dependencies:**
What must be in place before this epic can start?

**Initial Issues:**
Link to child issues that will be created.

**Milestone:** Phase N: ...
**Labels:** type:epic, P:x
```
